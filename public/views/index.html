Add a New Version

Use the form below to add a new version (e.g. after a release) to the publication cycle.

You will need 3 things for this.
A Version Number
This is the PrestaShop version number, like 1.6.0.10, 1.6.1.0, etc.
The version number must be the one that is displayed in the backoffice of the prestashop version you are adding.
Important: This version number must NOT already be in use on Crowdin.
E.g., if you're adding a development version, check that the version number was changed. If it is the same as that of a stable version, abort.
A Version Header
This is the integer id that identifies a version on translate.prestashop.com. They can be found under "PS Versions" on translate.prestashop.com.
To get a version header for a new version, you need to first add the version on translate.prestashop.com.
You need to add a version using the so called "by hand method", see the form on translate.prestashop.com.
A ZIP Archive
This is just a ZIP of PrestaShop. The PrestaShop inside must be in a folder called "prestashop".
It must have the admin and install folders named in a standard way: either admin or admin-dev, either install or install-dev.
When publishing a development version of PrestaShop, you need to zip the folder with git initialized, with everything set to track the branches you intend to use.
E.g.: prestashop on branch "release", all submodules on branch "master"
